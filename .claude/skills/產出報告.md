# 產出報告

用戶說「產出報告」「今天的報告」「produce report」時觸發。

## 步驟

### 1. 拉取最新

```bash
git pull origin main
```

### 2. 讀取近 7 天事件

```bash
for i in $(seq 0 6); do
  DATE=$(date -v-${i}d +%Y-%m-%d 2>/dev/null || date -d "-${i} days" +%Y-%m-%d)
  FILE="data/events/$DATE.jsonl"
  if [ -f "$FILE" ]; then
    COUNT=$(wc -l < "$FILE" | tr -d ' ')
    echo "$DATE: $COUNT 筆"
  fi
done
```

### 3. 識別重要事件

讀取事件檔，依 `importance.score` 排序，取 top 5：
- 關注 score > 0.8 的事件
- 標記涉及多家公司（供應鏈上下游同時提及）的事件
- 標記異常（anomalies 欄位非空）的事件

### 4. 讀取財務數據

```bash
cat data/financials/latest.json | python3 -c "
import json, sys
d = json.load(sys.stdin)
alerts = [c for c in d['companies'] if c.get('alert')]
print(f'財務警示: {len(alerts)} 家')
for a in alerts:
    print(f'  {a[\"name\"]}: AR {a.get(\"ar_qoq\",\"?\")}, 庫存 {a.get(\"inv_qoq\",\"?\")}')
"
```

### 5. 寫入 LLM 分析

讀取現有的 daily report JSON（由 Actions 自動產生），追加 `llm_analysis` 欄位：

```json
{
  "llm_analysis": {
    "summary": "2-3 句市場摘要",
    "signals": [
      { "text": "訊號描述", "level": "red|yellow|green" }
    ]
  }
}
```

**Signal levels:**
- `red`：需要立即關注（如：庫存暴增、重大供應鏈中斷）
- `yellow`：持續觀察（如：價格波動、產能調整）
- `green`：正面指標（如：需求成長、獲利超預期）

### 6. 寫入 7 日報告 LLM 分析

```json
{
  "llm_analysis": {
    "summary": "3-4 句週度摘要",
    "watchlist": [
      { "company": "公司名", "reason": "觀察原因" }
    ]
  }
}
```

### 7. Commit 並推送

```bash
git add site/data/reports/
git commit -m "report: daily analysis $(date +%Y-%m-%d)"
git push origin main
```

## 報告品質規範

- `summary` 必須根據實際事件撰寫，禁止憑記憶
- `signals` 必須引用具體事件（哪家公司、什麼事）
- 財務數據必須從 `latest.json` 讀取，不可猜測
- 保留既有 JSON 欄位，只追加 `llm_analysis`
