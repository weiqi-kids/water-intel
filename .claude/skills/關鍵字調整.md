# 關鍵字調整

用戶說「關鍵字審計」「gate2 擋太多」「漏掉重要新聞」「keyword audit」時觸發。

## 觸發條件

- `filter_audit.gate2_block_rate > 15%`
- 用戶反映某類新聞沒有出現在事件時間軸

## 步驟

### 1. 讀取 filter_audit

```bash
TODAY=$(date +%Y-%m-%d)
python3 -c "
import json
d = json.load(open('site/data/reports/daily/$TODAY.json'))
fa = d.get('filter_audit', {})
print(f'總處理: {fa.get(\"total_raw\", \"?\")}')
print(f'通過率: {fa.get(\"pass_rate\", \"?\")}')
print(f'Gate2 擋住率: {fa.get(\"gate2_block_rate\", \"?\")}%')
print()
for s in fa.get('gate2_samples', [])[:10]:
    print(f'  被擋: {s.get(\"title\", \"?\")[:80]}')
"
```

### 2. 判斷被擋文章

逐篇判斷是否與本追蹤產業相關：
- **相關** → 找出缺少的關鍵字，記錄應加入哪個 topic
- **不相關** → 正確擋住，無需處理

### 3. 呈現審計結果

```markdown
## 關鍵字審計結果

通過率：XX% | Gate2 擋住率：XX%

### 被擋但應通過的文章
| 標題 | 缺少的關鍵字 | 建議加入的主題 |
|------|-------------|--------------|
| ... | ... | ... |

### 建議新增關鍵字
topics.yml → {topic_id} → keywords 新增：
- keyword1
- keyword2
```

### 4. 更新 configs/topics.yml

用戶確認後，將新關鍵字加入對應 topic 的 `keywords` 陣列。

### 5. 驗證

```bash
git add configs/topics.yml
git commit -m "tune: add keywords to {topic_id}"
git push origin main
gh workflow run "Daily Data Ingest" -R $(gh repo view --json nameWithOwner -q .nameWithOwner)
```

等 pipeline 跑完後，再次檢查 gate2_block_rate 是否下降。

## 注意

- 不要加太泛的關鍵字（如 "technology"），會導致大量不相關文章通過
- 每次調整後必須在 GitHub Actions 跑一次驗證
- `configs/topics.yml` 中每個 topic 也有 `sentiment_keywords`（正面/負面），這裡不調整那部分
