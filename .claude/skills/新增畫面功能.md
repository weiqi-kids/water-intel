# 新增畫面功能

用戶說「Dashboard 加一個 XX」「新增功能」時觸發。

## REQUIRED BACKGROUND

修改 `site/index.html` 前，**必須先讀「畫面規範」skill**，確保新增元件符合 design tokens。

## 開發流程

### 1. 設計（brainstorming）

使用 superpowers:brainstorming skill：
- 確認功能需求
- 確認放在畫面哪個位置
- 確認資料來源（現有 JSON 還是需要新增後端）
- 寫 spec 並審查

### 2. 實作規劃（writing-plans）

使用 superpowers:writing-plans skill：
- 拆分成可獨立 commit 的小 task
- 每個 task 明確列出修改檔案和程式碼

### 3. 執行（subagent-driven-development）

使用 superpowers:subagent-driven-development skill：
- 每個 task 派 subagent 實作
- 每個 task 做 spec review + code quality review

### 4. 驗證

```bash
git push origin main
gh workflow run "Daily Data Ingest" -R $(gh repo view --json nameWithOwner -q .nameWithOwner)
# 等完成後開 （本專案 GitHub Pages URL） 驗證
```

## 新增 tab 的標準步驟

1. HTML：在 `#stock-tabs` 加 tab button，加 view container
2. CSS：如需新樣式，使用 CSS variables（見畫面規範）
3. JS：
   - 在 tab click handler 的 view list 加入新 view name
   - 實作 `renderNewView()` 函數
   - 在 `updateRangeButtons()` 加入新 tab 的 range 按鈕（如需要）
   - 在 `onBrushEnd()` 加入呼叫（如需連動 brush）
4. 測試：本地 `python3 -m http.server 6229 -d site` 驗證

## Brush 連動規則

| 計算類型 | 觸發時機 |
|---------|---------|
| 輕量（走勢圖、時間軸） | brush 拖動中（即時） |
| 重量（熱力圖、回測、供應鏈上色） | brush 放開（`event.type === 'end'`） |
| 分布計算 | 按鈕點擊 |

## 可用的計算函數

- `calcReturn(companyId, start, end)` — 報酬率
- `calcMDD(companyId, start, end)` — 最大回撤
- `calcDistribution(companyId, months)` — 報酬率分布
- `calcRecovery(companyId, months)` — 回撤恢復
- `calcFundFlow(etfId, start, end)` — 資金流向
- `findNearestTradingDay(prices, target, 'after'|'before')` — 日期對齊
- `companyLink(companyId, displayName)` — 公司超連結
- `ensureFinancialsHistory()` — 延遲載入財務歷史
- `ensureHoldersHistory()` — 延遲載入持倉歷史
