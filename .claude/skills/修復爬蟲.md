# 修復爬蟲

用戶說「fetcher 壞了」「XX 抓不到新聞」「修復爬蟲」時觸發。

## 診斷步驟

### 1. 找出失敗的 fetcher

```bash
RUN_ID=$(gh run list -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --workflow="Daily Data Ingest" --limit 1 --json databaseId -q '.[0].databaseId')
JOB_ID=$(gh run view $RUN_ID -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --json jobs -q '.jobs[0].databaseId')
gh run view --job=$JOB_ID -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --log 2>/dev/null | grep "ERROR:fetchers" | sed 's/.*ERROR:fetchers[^:]*://' | sort
```

### 2. 分類錯誤

| 錯誤類型 | 解法 |
|----------|------|
| 403 Forbidden | 網站擋爬蟲 → 找 RSS feed 或改 playwright |
| 404 Not Found | URL 失效 → 找正確 URL |
| SSL Error | 憑證問題 → 改 playwright |
| Timeout | 網路限流 → 增加 http_timeout 或偶發性可忽略 |
| 500 Server Error | 目標網站問題 → 找替代 URL |

### 3. 研究替代來源

依序嘗試：
1. **IR RSS feed** — 最穩定，用 WebSearch 搜 `{company} investor relations rss`
2. **新聞頁面 HTTP** — 用 WebFetch 測試 URL 是否回 200
3. **Playwright** — 最後手段，JS 渲染的頁面

常見 RSS 路徑：
- `https://ir.{company}.com/rss/news-releases.xml`
- `https://investor.{company}.com/rss-news-feed`
- `https://{company}.com/feed/`

## 修復步驟

### 4. 更新兩個地方

**a) `configs/companies.yml`** — 更新對應公司的欄位：
- `ir_rss_url`: RSS feed URL（如果找到）
- `news_url`: 新聞頁面 URL
- `fetch_mode`: `rss` / `http` / `playwright`

**b) `fetchers/{company_id}.py`** — 更新 class 屬性：
- `news_url` — 新 URL
- `ir_rss_url` — RSS feed（如果有）
- `fetch_mode` — 對應的模式

注意：`scripts/fetch_companies.py` 會從 `companies.yml` 讀取 `fetch_mode` 和 `ir_rss_url` 覆蓋 fetcher class 的值。兩邊都要改。

### 5. 驗證

```bash
git add configs/companies.yml fetchers/*.py
git commit -m "fix: update {company} fetcher URL"
git push origin main
gh workflow run "Daily Data Ingest" -R $(gh repo view --json nameWithOwner -q .nameWithOwner)
# 等 5-7 分鐘
RUN_ID=$(gh run list -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --workflow="Daily Data Ingest" --limit 1 --json databaseId -q '.[0].databaseId')
JOB_ID=$(gh run view $RUN_ID -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --json jobs -q '.jobs[0].databaseId')
gh run view --job=$JOB_ID -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --log 2>/dev/null | grep "ERROR:fetchers" | sed 's/.*ERROR:fetchers[^:]*://' | sort
```

確認修復的公司不再出現在錯誤列表中。

## 重要提醒

- 改完一定要在 GitHub Actions 上跑一次驗證，不能只本地測完就交差
- Timeout 類錯誤可能是偶發的（GitHub Actions IP 被限流），不一定需要修
- 改 `fetch_mode` 時，`companies.yml` 和 `fetchers/*.py` 兩邊都要改
