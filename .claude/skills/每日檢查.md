# 每日檢查

進入本 repo 或用戶說「今天狀態」「每日檢查」「daily check」時觸發。

## 執行步驟

### 1. 同步最新

```bash
git pull origin main
```

### 2. 檢查 Actions 狀態

```bash
gh run list -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --limit 3
```

如果最近一次失敗，查看失敗原因：
```bash
gh run view <run-id> --log-failed -R $(gh repo view --json nameWithOwner -q .nameWithOwner)
```

### 3. 檢查今日事件數

```bash
TODAY=$(date +%Y-%m-%d)
wc -l data/events/$TODAY.jsonl 2>/dev/null || echo "今日無事件檔"
```

### 4. 檢查 fetcher 錯誤

從最近一次 Actions 日誌中提取 ERROR：
```bash
RUN_ID=$(gh run list -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --workflow="Daily Data Ingest" --limit 1 --json databaseId -q '.[0].databaseId')
gh run view --job=$(gh run view $RUN_ID -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --json jobs -q '.jobs[0].databaseId') -R $(gh repo view --json nameWithOwner -q .nameWithOwner) --log 2>/dev/null | grep "ERROR:fetchers" | sed 's/.*ERROR:fetchers[^:]*://' | sort
```

### 5. 關鍵字審計

```bash
TODAY=$(date +%Y-%m-%d)
cat site/data/reports/daily/$TODAY.json 2>/dev/null | python3 -c "
import json, sys
d = json.load(sys.stdin)
fa = d.get('filter_audit', {})
print(f'通過率: {fa.get(\"pass_rate\", \"N/A\")}')
print(f'Gate2 擋住率: {fa.get(\"gate2_block_rate\", \"N/A\")}%')
if fa.get('alert'): print('!! 需要關鍵字調整')
"
```

如果 `gate2_block_rate > 15%`，建議執行「關鍵字調整」skill。

### 6. 輸出報告格式

```
每日狀態 — {date}
- Actions: [成功/失敗] (最近 3 次)
- 今日事件: N 筆
- Fetcher 錯誤: N 個 (列出公司名)
- 關鍵字審計: 通過率 XX% / Gate2 擋住率 XX%
- 需要處理: [列出問題]
```

## 後續動作

- Fetcher 錯誤 → 觸發「修復爬蟲」skill
- Gate2 擋住率高 → 觸發「關鍵字調整」skill
- Actions 失敗 → 查看日誌，判斷原因
